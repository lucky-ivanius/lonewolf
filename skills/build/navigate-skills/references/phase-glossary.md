# Phase glossary

Launch Kit groups skills into six phases. Each phase corresponds to a moment in the journey from "I want to build something" to "it's launched, people use it, and it earns".

## learn

Skills that teach the fundamentals. The user is new to building products, or new to the agent economy, and is building a mental model. Output is understanding, not code.

Examples: `founder-101`, `learn`.

Use when the user is asking conceptual questions, not asking for code yet.

## idea

Skills that help pick what to build. The user has the desire to build, but no chosen idea, or an idea that has not been validated. Output is a chosen idea with a write-back to `.lonewolf/idea-context.md`.

Examples: `find-next-idea`, `validate-idea`, `competitive-landscape`, `defi-market-research`.

Use before the user has scaffolded any code.

## build

Skills that produce code and pressure-test the product while it is being built. The bulk of the catalog: the intent loop, scaffolding, the MVP loop, agents, the design family, the anti-slop family, security.

Examples: `clarify-intent`, `plan-before-code`, `verify-against-intent`, `scaffold-project`, `build-mvp`, `build-ai-agent`, `brand-design`, `design-taste`, `frontend-design-guidelines`, `number-formatting`, `page-load-animations`, `product-review`, `roast-my-product`, `validate-business-model`, `retention-loop`, `will-real-users-pay`, `security-review`, `navigate-skills`.

Use when the user is writing code, reviewing their own product, or hardening it.

## ship

Skills that take working code into the world. Go-live gates, pitch decks, videos, grants.

Examples: `launch-checklist`, `create-pitch-deck`, `marketing-video`, `video-craft`, `apply-grant`.

Use when the project builds cleanly and the user is moving to a real environment or audience.

## grow

Skills that earn the first users, not just a launch tweet. Positioning, landing copy, distribution, manual first-user tactics.

Examples: `positioning`, `landing-copy`, `distribution-plan`, `first-100-users`.

Use when the project is live (or about to be) and the user is asking how to get users.

## earn

Skills that turn real users into real revenue, not vanity metrics. Pricing, revenue diagnosis, payment rails.

Examples: `pricing`, `revenue-review`, `get-paid`.

Use when users exist and the question is money.

## How to use this glossary

- If the user's request maps to one phase, recommend skills from that phase first.
- If the request straddles phases (common: build + ship, ship + grow), recommend the earlier-phase skill first, since later-phase skills usually depend on its output.
- If the user is mid-phase but hitting a meta question ("am I doing this right"), the anti-slop family in build (`validate-business-model`, `roast-my-product`, `product-review`) is often the right answer.
