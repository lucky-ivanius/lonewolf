# Anti-generic crypto brand checklist

If three or more of these fire, the brand will read as generic crypto and is worth redoing.

## Color markers

- Primary is purple, especially `#6F00FF`, `#8B5CF6`, or any near-Coinbase-purple.
- Background uses a black-to-purple radial gradient.
- Accent is neon green, neon cyan, or chrome silver.
- Buttons use a glow effect (`box-shadow` with a colored blur of 20px or more).
- Hero background has animated gradient noise or a particle field.

## Type markers

- Headings in default Inter, default system-ui, or default Helvetica without modification.
- Body in the same family as the heading at the same weight (no contrast).
- Numbers (token amounts, prices) in a tabular-figure-less proportional sans, making numbers jiggle on update.
- All caps headings with wide letter-spacing applied indiscriminately.

## Layout markers

- Hero section is a large headline plus a glow-effect button plus a "particles" or "wave" background canvas.
- "Trusted by" logos in a horizontal scroll, all desaturated to grey.
- Feature grid of three or six cards, each with a circular gradient icon at top-left.
- Footer with a CTA banner that says "Join the future" or similar.

## Copy markers

- Tagline is "the future of X" or "X, reimagined".
- Buttons say "Launch app" without context.
- Section headings are sentence-case but with random words capitalized.
- Marketing copy uses "leverage", "cutting-edge", "world-class", "revolutionary", or "AI-powered" (launchkit's banned list, but also a generic-brand signal).

## How to recover

When markers fire, two moves:

1. Replace the offending element with something specific to the product (e.g. swap purple-gradient for a single saturated primary in a real domain color).
2. Add one weird detail that no template would generate. A custom illustration, an unusual font pairing, a copy line that names the user's actual problem.

Generic does not mean bad. It means forgettable. The fix is specificity, not maximalism.
