# Milestone templates

Three shapes by grant length. Each milestone bundles deliverables, sets a date, and ties a payout. The reviewer reads milestones as the trust contract.

## 3 month grant

- M1 (month 1): scoped MVP deployed to testnet. Deliverables: testnet package, basic frontend, repo public. Payout: 30%.
- M2 (month 2): integrations and validation. Deliverables: load-bearing sponsor integration shipped, 5 to 10 candidate user signals captured. Payout: 30%.
- M3 (month 3): mainnet launch and writeup. Deliverables: mainnet package, docs, post-mortem of the grant period. Payout: 40%.

## 6 month grant

- M1 (month 1): scope locked, testnet MVP. Deliverables: design doc, testnet package, public repo. Payout: 15%.
- M2 (month 2): core features. Deliverables: feature 1, feature 2 (named in proposal). Payout: 20%.
- M3 (month 3): integrations and partnerships. Deliverables: load-bearing sponsor integration, one partner integration. Payout: 20%.
- M4 (month 4): user testing and iteration. Deliverables: user study with 10+ participants, iteration log. Payout: 15%.
- M5 (month 5): security review and audit prep. Deliverables: `security-review` complete, P0 items resolved or accepted. Payout: 15%.
- M6 (month 6): mainnet and post-launch. Deliverables: mainnet package, post-mortem, sustainability plan execution started. Payout: 15%.

## 12 month grant

- M1 (month 1): scope locked. 10%.
- M2 (month 2): testnet MVP. 10%.
- M3 (month 3): core features 1. 10%.
- M4 (month 4): core features 2. 10%.
- M5 (month 5): integrations. 10%.
- M6 (month 6): mid-term review. 10%.
- M7 (month 7): user testing. 5%.
- M8 (month 8): security review. 5%.
- M9 (month 9): mainnet. 10%.
- M10 (month 10): post-launch iteration. 5%.
- M11 (month 11): documentation and ecosystem hand-off. 5%.
- M12 (month 12): final report and sustainability check-in. 10%.

## Definition-of-done rules per milestone

- The deliverable must be externally verifiable: a public commit, a deployed package id, a published doc, a recorded user study.
- "We worked on X" is not a deliverable. "X is shipped at <URL>" is.
- Milestones never depend on private data.

## What this template is NOT

- Not a one-size-fits-all schedule. Adapt to the project's actual scope and risks.
- Not a frozen contract. If a milestone slips, communicate before the date and propose an amended schedule.
- Not a ban on flexibility. The grant program can negotiate.
